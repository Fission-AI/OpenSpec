# Apply Skill — Test Integration

**Question:** Should the apply skill run tests after implementing tasks, and write tests for the changes it makes?

## Context

The `enpalspec:apply` skill implements tasks from an OpenSpec change. Currently it focuses on
writing implementation code, but doesn't run tests or author tests for the changes it introduces.

Two distinct ideas bundled in the prompt:
1. **Run tests after implementing** — verify the implementation didn't break anything
2. **Write tests for changes** — add new test coverage for the new behavior
