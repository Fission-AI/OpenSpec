## ADDED Requirements

### Requirement: Skill and CLI Entry-Point Guidance
The generated `openspec/AGENTS.md` SHALL name the available OpenSpec skills and the core deterministic CLI commands and state when to use each, so an agent that reads only `AGENTS.md` — without loading `SKILL.md` skills — still drives the OpenSpec workflow correctly.

#### Scenario: Skills advertised
- **WHEN** `openspec/AGENTS.md` is generated or updated
- **THEN** it SHALL list the workflow skills (for example propose, continue, apply, verify, archive) and the user-facing condition under which each applies
- **AND** it SHALL direct skill-loading agents to invoke the matching skill rather than improvising the workflow

#### Scenario: Deterministic CLI advertised
- **WHEN** `openspec/AGENTS.md` is generated or updated
- **THEN** it SHALL name the core read-and-validate commands the workflow depends on (for example `openspec list`, `openspec status`, `openspec instructions`, `openspec validate`)
- **AND** it SHALL instruct agents to rely on their JSON output for paths and state rather than assuming repository-local paths

#### Scenario: Generated artifacts are managed
- **WHEN** `openspec/AGENTS.md` describes the generated skill and command files
- **THEN** it SHALL state that those files are regenerated by `openspec update` and SHALL NOT be hand-edited
