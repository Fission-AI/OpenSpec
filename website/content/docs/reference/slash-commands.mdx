Slash commands run in your **AI assistant's chat**, not your terminal. The exact punctuation varies by tool (`/opsx:propose` in Claude Code, `/opsx-propose` in Cursor, and so on) — see [Supported Tools](/docs/reference/supported-tools). Below they are written in the Claude Code style.

## Core profile

These five ship by default. They cover the entire loop.

### `/opsx:explore`

Your thinking partner. Investigates the codebase, compares options, draws diagrams, and sharpens a fuzzy idea into a concrete plan. Creates no files and writes no code. Hands off to `/opsx:propose` when you are ready. [Full guide.](/docs/explore)

### `/opsx:propose`

```text
/opsx:propose <change-name>
```

Creates a new change and drafts all of its planning artifacts at once — proposal, spec deltas, design, and tasks. This is your review moment before any code is written.

### `/opsx:apply`

Implements the change. Works through `tasks.md`, writes code, and checks off each item. Resumes from the first unchecked task if you come back later.

### `/opsx:sync`

Merges a change's delta specs into the main specs. Optional — `/opsx:archive` will prompt you when a sync is needed, so you rarely call this directly.

### `/opsx:archive`

Archives a completed change: merges its deltas into the source of truth and moves the folder to `openspec/changes/archive/YYYY-MM-DD-<name>/`.

## Expanded profile

Enable these with `openspec config profile`, then `openspec update`. They give you finer-grained control for bigger or more deliberate work.

### `/opsx:new`

```text
/opsx:new <change-name>
```

Scaffolds a change — creates the folder and metadata and shows the first artifact template — without drafting all artifacts yet.

### `/opsx:continue`

Creates the next artifact in the dependency chain, one at a time. Good for incremental, deliberate progress.

### `/opsx:ff`

Fast-forward. Creates all remaining planning artifacts at once, in dependency order. The expanded-profile equivalent of letting `propose` draft everything.

### `/opsx:verify`

Checks that your implementation matches the change's artifacts — completeness, correctness, and coherence. Reports issues without blocking archive. A fast way to surface code/spec drift.

### `/opsx:bulk-archive`

Archives several completed changes at once, handling any spec conflicts between them.

### `/opsx:onboard`

A guided, interactive tutorial that runs the full workflow on your actual codebase — it finds a small, safe improvement and walks you through proposing, building, and archiving it.

<Callout title="Two profiles, one loop">
Most people only ever need the core five. The expanded set exists for when you want to create artifacts step by step or verify before archiving. See [The Workflow](/docs/the-workflow) for how they fit together.
</Callout>
