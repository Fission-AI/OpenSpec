OpenSpec can write artifacts in any language. There is no special setting — you just tell the AI which language to use through your project context, and every generated proposal, spec, and task list follows.

## Quick setup

Add a language instruction to `openspec/config.yaml`:

```yaml
schema: spec-driven

context: |
  Language: Portuguese (pt-BR)
  All artifacts must be written in Brazilian Portuguese.

  # ...the rest of your project context
  Tech stack: TypeScript, React, Node.js
```

That is it. Every artifact OpenSpec generates is now in Portuguese.

## Keep technical terms in English

A common preference — write prose in your language, but leave API names, code, and paths untouched:

```yaml
context: |
  Language: Japanese
  Write in Japanese, but:
  - Keep technical terms like "API", "REST", "GraphQL" in English
  - Code examples and file paths remain in English
```

## Verify it is working

```bash
openspec instructions proposal --change my-change
```

The output includes your language context, so you can confirm it is being injected before you generate anything.

<Callout title="Why this works">
Anything in `config.yaml`'s `context:` is added to every planning request the AI receives. Language is just one kind of context — it sits alongside your stack, conventions, and rules. See [Customization](/docs/customization).
</Callout>
