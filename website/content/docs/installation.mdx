OpenSpec is a small command-line tool. Install it once, globally, and use it in any project.

<Callout title="Requirements">
OpenSpec needs **Node.js 20.19.0 or higher**. Check yours with `node --version`.
</Callout>

## Install the CLI

<Tabs items={['npm', 'pnpm', 'yarn', 'bun']}>
  <Tab value="npm">
    ```bash
    npm install -g @fission-ai/openspec@latest
    ```
  </Tab>
  <Tab value="pnpm">
    ```bash
    pnpm add -g @fission-ai/openspec@latest
    ```
  </Tab>
  <Tab value="yarn">
    ```bash
    yarn global add @fission-ai/openspec@latest
    ```
  </Tab>
  <Tab value="bun">
    ```bash
    bun add -g @fission-ai/openspec@latest
    ```

    Bun installs OpenSpec, but OpenSpec runs on Node.js 20.19.0+, which must be available on your `PATH`.
  </Tab>
</Tabs>

## Verify it worked

```bash
openspec --version
```

If you see a version number, you are ready. Next stop: [Getting Started](/docs/getting-started).

## Nix

Prefer Nix? You can run OpenSpec without installing it:

```bash
nix run github:Fission-AI/OpenSpec -- init
```

Or add it to your profile:

```bash
nix profile install github:Fission-AI/OpenSpec
```

Or pull it into a dev shell in your `flake.nix`:

```nix
{
  inputs = {
    nixpkgs.url = "github:NixOS/nixpkgs/nixos-unstable";
    openspec.url = "github:Fission-AI/OpenSpec";
  };
  outputs = { nixpkgs, openspec, ... }: {
    devShells.x86_64-linux.default =
      nixpkgs.legacyPackages.x86_64-linux.mkShell {
        buildInputs = [ openspec.packages.x86_64-linux.default ];
      };
  };
}
```

## Keeping OpenSpec current

Upgrade the package the same way you installed it:

```bash
npm install -g @fission-ai/openspec@latest
```

Then, inside each project, refresh the generated AI instructions so your assistant has the latest slash commands:

```bash
openspec update
```

See [How Commands Work](/docs/how-commands-work) for what `openspec update` actually regenerates.
