OpenSpec has two halves, and almost everyone trips over the boundary once. Read this page and you will not.

## The two halves

<Cards>
  <Card title="The CLI — your terminal" icon="Terminal">
    `openspec ...` commands run in your shell. They set up the project, list and validate changes, show a dashboard, and archive work.
  </Card>
  <Card title="Slash commands — your AI chat" icon="MessageSquare">
    `/opsx:...` commands run in your AI assistant's chat box. They tell the AI to follow the workflow: draft a proposal, write specs, build from tasks, archive.
  </Card>
</Cards>

A quick way to remember it:

```text
TERMINAL   openspec init        openspec list      openspec view
AI CHAT    /opsx:explore        /opsx:propose      /opsx:apply
```

If you type `/opsx:propose` into your terminal, nothing happens — that command belongs in chat. If you type `openspec init` into your AI chat, you are just asking the AI to run a shell command for you, which is not the point.

## Why the split exists

The CLI is the **engine**. It knows the rules everywhere: what a change folder looks like, how artifacts depend on each other, how spec deltas merge. It is the same no matter which AI tool you use.

The slash commands are the **steering wheel**. Each AI tool has its own syntax and mechanics, so `openspec init` generates the right kind of command file for whichever tool you are using. That is why setup happens in the terminal but the day-to-day driving happens in chat.

## There is no "interactive mode"

There is no separate mode to enter and no button to press. You simply:

<Steps>
<Step>Open your AI assistant — Claude Code, Cursor, Windsurf, Codex, and so on.</Step>
<Step>Type a slash command in the chat, like `/opsx:propose`.</Step>
<Step>The AI recognizes it, loads the matching OpenSpec instructions, and starts the workflow.</Step>
</Steps>

That is it. The slash command itself kicks off the work.

## Slash command syntax varies by tool

The exact punctuation differs between assistants. A few examples:

| Tool | How you invoke propose |
|------|------------------------|
| Claude Code | `/opsx:propose` |
| Cursor | `/opsx-propose` |
| Windsurf | `/opsx-propose` |
| GitHub Copilot | `/opsx-propose` |
| Trae | `/openspec-propose` |

When OpenSpec docs write `/opsx:propose`, read it as "the propose command, in whatever form your tool uses." See [Supported Tools](/docs/reference/supported-tools) for the full list.

## Confirming it is installed

If your slash commands are not showing up:

- Type `/opsx` in your AI chat and check the autocomplete.
- Look for generated files such as `.claude/skills/openspec-*`.
- Re-run `openspec update` in the project to regenerate the command files for your installed version.

<Callout type="info">
`openspec view` is a **dashboard viewer**, not where you propose or build changes. Proposing and building happen with slash commands in your AI chat.
</Callout>
